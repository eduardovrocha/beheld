# beheld v0.4.0 — Refundação multi-tool

**Release date:** 2026-06-02
**Tag:** `v0.4.0`
**Type:** Major release (no breaking changes for end users; one internal-API breaking change for custom scorer authors)

---

## TL;DR

Git history is now the spine of your Beheld profile. Eight coding harnesses contribute as additive enrichment. One command (`beheld harness install`) sets up every harness Beheld detects on your machine — no more tracking down which CLI to run for each tool.

```sh
curl -fsSL https://beheld.dev/install | sh
beheld harness install
beheld view
```

## What's new for you

### Eight harnesses supported

Beheld now observes:

| Harness | Capture fidelity | Status |
|---|---|---|
| Claude Code | `native_hook` | ✅ shipping |
| Continue.dev | `editor_extension` | ✅ shipping |
| Windsurf · Cascade Hooks | `native_hook` | ✅ shipping |
| Cursor | `local_log_tail` | ✅ shipping |
| Copilot CLI | `statusline` | ✅ shipping |
| Copilot VS Code | `local_log_tail` | ✅ shipping |
| Gemini CLI | `native_hook` | ⚠ manual setup (upstream hook spec not yet stable) |
| Codex CLI | `native_hook` | ⚠ manual setup (upstream hook spec not yet stable) |

Each session in your `.dpbundle` declares which harness contributed it and at what fidelity, so a recruiter inspecting your bundle can see exactly where each signal came from. The closed `capture_fidelity` enum — `native_hook`, `editor_extension`, `local_log_tail`, `statusline`, `inferred` — is the contract.

### `beheld harness install` — one command, all harnesses

```
beheld harness list      # show every harness + detection state
beheld harness install   # install hooks / enable tails for everything detected
beheld harness install windsurf cursor   # only specific harnesses
beheld harness install --force           # install even if not detected
```

The installer:

- **Writes hooks** for Claude Code, Continue.dev, and Windsurf (`~/.codeium/windsurf/hooks.json` with 12 Cascade entries).
- **Enables tails** for Cursor, Copilot CLI, and Copilot VS Code by writing one line to `~/.beheld/config.json`. The daemon picks them up on the next start and polls each one every 60 seconds.
- **Prints manual instructions** for Gemini CLI and Codex CLI until their hook specs stabilise — no speculative config files written.

Idempotent: re-running reports `already_migrated` for unchanged adapters.

### L1-first onboarding (`npx beheld`)

The default behavior changed: running `beheld` with no arguments now triggers `bootstrap` automatically when you don't have signing keys yet. The wizard sets up `~/.beheld/`, migrates `~/.devprofile/` non-destructively (see below), and points you at the next steps.

Already-installed users see the regular help — nothing changed for them.

### Non-destructive `~/.devprofile/` → `~/.beheld/` migration

Upgrading from pre-Refundação? The bridge now **copies** files instead of moving them and writes `~/.devprofile/MIGRATED_TO_BEHELD.md` as proof. Your old directory stays put as a rollback path; remove it manually with `rm -rf ~/.devprofile` when you've confirmed everything works.

A re-run reports `already_migrated` and short-circuits without re-copying.

### Portal — Capture Sources block on `/v/:id`

When someone visits your profile page, they see a new block at the bottom of the L2 section listing every harness that contributed, with the fidelity badge color-coded by trust tier:

- 🟢 `native_hook` / `editor_extension` — high trust
- 🟡 `local_log_tail` / `statusline` — medium trust
- 🔴 `inferred` — low trust (flagged visibly)

aria-labels carry the per-tier explanation so screen readers get it too.

## What's new under the hood

### Bundle schema v7

- `payload.scores.{prompt_quality, growth_rate, overall}` widened to `Optional[int] | null` in canonical JSON. When a dimension genuinely has no input (e.g. PromptQuality without enrichment, GrowthRate with <6 months of history), the score is `null` — never a fabricated neutral value. Honors the "honestidade de captura" principle.
- `payload.l1` / `payload.l2` renamed to `payload.core` / `payload.enrichment`. The old names are recognised by the **reader** as `v6_legacy` / `v5_legacy` so existing bundles still verify; the **generator** emits v7 only.
- `payload.enrichment.harness_sources[]` is first-class: every entry carries `(harness, capture_fidelity, sessions)` and the array is sorted lexicographically by `(harness, capture_fidelity)` for canonical-byte stability.

### Verifier fallback chain

```
v7 → v6_legacy → v5_legacy → v1_legacy → unknown
```

Bundles from any prior Beheld version verify offline with `beheld verify`.

### Portal schema v3

The portal (`beheld-web`) accepts wire BUNDLE_VERSION 6 **or** 7 under the same `schema_version: "v3"` label. Snapshot model detects v3 before v2 (defensive against hybrid payloads). The `BundleSignals` typed value object centralises the `core → l1 → empty` fallback chain so matchers and evolution curves never read raw `bundle_data`.

### GrowthRateScorer rewrite (spec §7.2)

Windows + signals + weights are now literal contract:

- ≥ 18 months of history → 12-month baseline + 6-month current (canonical).
- 6–18 months → 50/50 split with `confidence: low`.
- < 6 months → `null` (dimension absent from the profile rather than zeroed).
- Pesos canônicos: `ecosystems × 0.30 + platforms × 0.20 + test_ratio × 0.25 + diversity × 0.25` over signals normalized to [-1, +1].

`fallback_when_enrichment_missing=True` means GrowthRate computes from L1 alone when L2 is absent — a legitimate L1-only profile gets a real score, not a neutral 50.

## Breaking changes

**For end users — none.** Existing bundles verify, existing portals serve them, existing `beheld init` keeps working.

**For custom scorer authors** (internal API): scorers must now declare `data_sources` using `"core"` / `"enrichment"` instead of the pre-Refundação `"l1"` / `"l2"`. The `DataSource` Literal is re-exported from `packages/engine/src/scorers/base.py`. See `prompt_quality.py:28-29` for a one-line example.

## Privacy & security invariants (unchanged)

- Every adapter sanitises events at the handler boundary — secrets (`sk-*`, `ghp_*`, Bearer tokens, env values, passwords) never reach disk.
- Prompt text and response markdown are dropped at the handler boundary — only character counts (`prompt_length`, `response_length`) survive.
- `cwd` / `workspace` are SHA-256 hashed; raw paths never leave the handler.
- `~/.beheld/` permissions enforced at `0700`; `config.json` written at `0600`.
- Cross-language byte lock pinned in both Python and TypeScript prevents silent canonical-form drift.

## Upgrade path

### From any previous version

```sh
curl -fsSL https://beheld.dev/install | sh
beheld bootstrap          # idempotent — migrates legacy data if present
beheld harness install    # wire every harness Beheld detects
beheld start              # restart the daemon so it picks up the new tail config
```

If you had a `~/.devprofile/` directory from a pre-0.3 install, the bridge copies it to `~/.beheld/` and leaves the original in place. Verify everything works, then optionally `rm -rf ~/.devprofile` when ready.

### Custom scorer authors

Search-replace in your custom scorer files:

```python
# Before
data_sources: ClassVar[list[str]] = ["l1", "l2"]

# After
data_sources: ClassVar[list[DataSource]] = ["core", "enrichment"]
```

Add the import from `scorers.base`:

```python
from scorers.base import DataSource
```
