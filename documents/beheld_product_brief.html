Product brief do Beheld — definição de produto, ambientes e modelo de consentimento

Beheld

Perfil de desenvolvedor gerado silenciosamente a partir do uso real do Claude — com consentimento explícito e dados sob controle do usuário.

posicionamento

VS Code extension CLI global MCP server local plataforma web opcional
instalação ext. VS Code + npm i -g beheld
ambientes MVP Claude Code · VS Code + Continue.dev
armazenamento local-first · sync opt-in
modo de operação silencioso + consentimento explícito
monetização freemium — sync e benchmarks pagos

princípios de privacidade

1

Conteúdo nunca sai da máquina

Apenas sinais técnicos anonimizados são analisados. O texto das conversas fica 100% local.

2

Opt-in explícito na primeira execução

Wizard de onboarding mostra exatamente o que é coletado antes de qualquer análise.

3

Sync é sempre opt-in, nunca padrão

Perfil local funciona completo offline. A conta na plataforma é opcional e reversível.

4

Direito ao esquecimento via CLI

beheld delete --all apaga local e remoto sem suporte.

ambientes suportados no MVP

Claude Code MVP

Captura via hook PreToolUse / PostToolUse registrado no ~/.claude/settings.json. Acessa tool calls, bash commands, file edits e duração de sessão.

hooks nativos JSONL local sem extensão
VS Code + Continue.dev MVP

Extensão VS Code registra eventos via vscode.workspace API e intercepta chamadas do Continue.dev via MCP server local na porta 7337.

VS Code API MCP local extensão VSIX
Claude.ai (browser) v2

Extensão Chrome/Firefox intercepta respostas via fetch hook no service worker. Requer permissão explícita de leitura do domínio claude.ai.

browser ext. fetch hook alta fricção
Cursor v2

Cursor expõe API de extensão compatível com VS Code. O mesmo VSIX do MVP deve funcionar com adaptações mínimas no listener de eventos.

VS Code compat. reuso do VSIX

fluxo de consentimento (onboarding)

1 — Instalação detectada

Extensão VS Code abre painel de boas-vindas automaticamente. CLI exibe wizard interativo no primeiro beheld init.

2 — Transparência total

Lista exata do que é coletado, o que nunca é coletado, e onde fica armazenado. Sem letras pequenas.

3 — Opt-in granular

Usuário escolhe quais dimensões ativar: padrões de teste, plataformas, horários, tipos de projeto. Cada toggle independente.

4 — Coleta inicia (modo silencioso)

Ícone discreto na status bar do VS Code. Nenhuma notificação durante o trabalho. Badge atualiza o score ao fim de cada sessão.

5 — Sync (opt-in separado)

Convite para criar conta na plataforma web apenas após 7 dias de uso local. Nunca no onboarding inicial.

o que é e não é coletado

comandos bash executados coletado
nomes de arquivo e paths coletado
dependências detectadas coletado
timestamp e duração de sessão coletado
ferramentas MCP utilizadas coletado
texto das conversas com Claude nunca
conteúdo de arquivos editados nunca
variáveis de ambiente / secrets nunca
dados de negócio / domínio nunca
identidade / dados pessoais nunca
