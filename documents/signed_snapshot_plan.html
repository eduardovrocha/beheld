Planejamento do Signed Snapshot — componentes, fases e fluxo

Signed Snapshot — planejamento

Prova de competência verificável sem expor código, prompts ou PI. 100% local-first.

estrutura do .beheld

payload

scores, métricas, data, versão, previous_hash

hash

SHA-256 do payload serializado

signature

Ed25519 da chave privada em ~/.beheld/keys/

public_key

embutida — receptor verifica sem servidor

fases de implementação

1

Geração de chaves Ed25519

packages/cli Python/cryptography

Gerada automaticamente no beheld init se não existir. Opção de importar chave existente via beheld keys import. Armazenada em ~/.beheld/keys/private.pem com permissão 0600 — apenas o dono lê. Chave pública exportável em public.pem.

2

Chain of snapshots

packages/engine SQLite

Tabela snapshots no SQLite local. Cada snapshot referencia o hash do anterior (previous_hash). Primeiro snapshot tem previous_hash: null (genesis). Adulteração retroativa quebra toda a cadeia — detectável na verificação.

3

Geração do .beheld

packages/cli novo comando

beheld snapshot — monta o payload JSON com métricas autorizadas, calcula SHA-256, assina com Ed25519, embute a chave pública e salva o arquivo .beheld. Formato JSON com campos: version, payload, hash, signature, public_key.

4

Upload e QR code

plataforma web opt-in

beheld snapshot --share — faz upload do bundle para o portal, recebe uma URL curta (ex: beheld.dev/v/abc123) e gera o QR code no terminal via unicode block chars. Sem conta obrigatória — URL pública por 90 dias por padrão.

5

Verificação pública

página web sem conta

Página beheld.dev/v/abc123 exibe o perfil verificado. Receptor vê: scores, data do snapshot, status da assinatura (✓ válida / ✗ adulterada), integridade da chain. Também verificável offline via beheld verify arquivo.beheld.

novos comandos CLI

Geração

beheld snapshot — gera .beheld localmente

beheld snapshot --share — gera + upload + QR

beheld snapshot --output <path> — salva em path específico

beheld snapshot list — histórico de snapshots

Chaves e verificação

beheld keys show — exibe a chave pública atual

beheld keys import <path> — importa chave existente

beheld keys rotate — gera novo par, mantém histórico

beheld verify <arquivo.beheld> — verifica offline

impacto na stack existente

packages/engine (Python)

Nova tabela snapshots no SQLite. Endpoint POST /snapshot — monta payload e retorna dados para o CLI assinar. Lib: cryptography (já no ecossistema Python).

packages/cli (TypeScript)

Novos comandos: snapshot e keys. Geração Ed25519 via @noble/ed25519 (zero deps, auditável). QR code via qrcode-terminal.

plataforma web (v1)

Endpoint POST /bundles para upload. Página pública /v/:id para verificação. Verificação Ed25519 no browser via Web Crypto API — sem backend.
